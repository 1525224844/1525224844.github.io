sql注入的原理分析

注入攻击的本质：是把用户输入的数据当作代码执行
sql注入：针对sql语句的注入，也可以理解为用户输入的数据当作sql语句的代码执行

两个关键条件：

  • 第一是用户能够控制输入
  • 第二是原本程序要执行的代码，拼接了用户输入的数据

显错注入-联合查询

联合查询是拼接在原查询语句下，要求结果数量与原结果数量相同。

select * from security.users where id=1 union select 1,2,3;

1,2,3并无特殊含义，换成其他数字或字符串也可以.
